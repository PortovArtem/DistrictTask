**Описание проекта**

Веб-сервис предназначен для управления районом и решения задач районной активности. Приложение позволяет:

- авторизовывать пользователей и распределять их по ролям: руководитель, заместители, сотрудники;

- вести состав людей, назначать задачи и отслеживать их выполнение;

- контролировать прогресс через панель управления с возможностью внесения изменений;

- получать уведомления о новых задачах и приближении дедлайнов;

- наглядно отображать статус задач (выполнено, начато, просрочено).


Фронтенд реализован на React, бэкенд на Django, база данных — PostgreSQL. Репозиторий проекта ведётся в GitLab.

**Для пользователей**

- Доступ к приложению через браузер по ссылке: https://<ваш_сервис>.ru

- Авторизация и регистрация пользователей

- Создание и редактирование задач

- Назначение задач участникам по ролям

- Отслеживание статусов задач (начато, выполнено, просрочено)

- Получение уведомлений о новых задачах и приближении дедлайнов

- Панель управления с визуализацией прогресса задач


> Пользователю не нужно подключать базу данных или запускать сервер. **_Всё уже работает через веб._**

**Для разработчика**
Используемое ПО

- Python 3.9+ (Django)

- JavaScript (React)

- PostgreSQL

- Django REST Framework, Axios, React Router


Среда разработки: VS Code / PyCharm / WebStorm

_Инструкция по локальному запуску_

1. Клонировать репозиторий:

> git clone <URL_репозитория>


2. Установить зависимости для бэкенда:

> pip install -r requirements.txt


3. Настроить подключение к PostgreSQL в settings.py:

> DATABASES = {
>     'default': {
>         'ENGINE': 'django.db.backends.postgresql',
>         'NAME': '<имя_базы>',
>         'USER': '<пользователь>',
>         'PASSWORD': '<пароль>',
>         'HOST': 'localhost',
>         'PORT': '5432',
>     }
> }



4. Применить миграции:

> python manage.py migrate


5. Запустить сервер Django:

> python manage.py runserver


6. Установить зависимости фронтенда:

> cd frontend
> npm install



7. Запустить React-приложение:

> npm start


8. Открыть браузер и перейти по адресу: _http://localhost:3000_

> В локальном запуске разработчику нужна база данных района (например, тестовая база с пользователями и задачами).
